❌ 19: enum, companion object | Задачи

Когда мы создаем объекты класса, каждый получает свое собственное "место в памяти" для хранения данных. Если у тебя есть класс User со свойством name, то у каждого пользователя будет своя переменная name – Анна, Петр, Мария. Они хранятся отдельно.

Статические элементы работают иначе – они существуют в единственном экземпляре для всего класса. Это как общая доска объявлений в офисе: она одна на всех сотрудников, а не у каждого своя копия.

В Java это static, в Kotlin – companion object. Такие элементы "привязаны" к самому классу, а не к его объектам, поэтому доступны без создания экземпляров.

Companion object идеален для фабричных методов создания объектов, констант и утилит, которые нужны всему классу, а не отдельным объектам.
