Зачем нужны ветки?

Git создавался Линусом Торвальдсом преимущественно для работы над ядром Linux. Тысячи программистов присылали свои изменения, которые могли конфликтовать друг с другом (это когда одна и та же строка кода редактируется разными людьми). Некоторые изменения ломали Linux, и это выяснялось уже позже, в таком случае приходилось откатывать изменения.

Теперь представь, что задача которую ты делаешь решена неправильно, откатить ее будет сложно, если ты все делал в одной ветку, еще хуже — в одном коммите.

Другая проблема, это когда через пару месяцев работы над проектом другой разработчик пытается выяснить природу какой-то конкретной строки кода. Он смотрит коммит который принес эту строчку и видит "решил 5 задач". На то, чтобы вникнуть в контекст уйдет больше времени, чем если бы там было написано "Реализовал решение для задачи 4,  в которой требовалось отправить человека в космос".
