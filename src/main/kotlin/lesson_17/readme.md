Все таки поля или свойства?

Технически в Kotlin нет полей – все является свойствами (их еще называют property). Но в некоторых источниках называть их могут и полями. Либо от недопонимания, либо для простоты изложения.

Вот несколько источников по этой теме:

- https://kotlinlang.ru/docs/properties.html – общая информация по свойствам из официальной документации.

- https://habr.com/ru/companies/otus/articles/740910/ – про подкапотную генерацию геттеров и сеттеров.

- https://blog.trukhin.com/свойства-и-поля-в-kotlin-9f606e4544fb – хорошая статья на Medium. Используй VPN, чтобы открыть статью из РФ.
