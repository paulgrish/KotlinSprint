Конвенция Kotlin. Для автоматического форматирования кода в IDEA используется такое сочетание клавиш. Не забывай применять автоформат перед коммитом.

Ctrl (Cmd на маке) + Alt + L

Зачем вообще соблюдать стиль? Ну во-первых есть такая штука, как Coding conventions - https://kotlinlang.org/docs/coding-conventions.html, определенная разработчиками языка. Руководства созданы, чтобы облегчить чтение и понимание кода, улучшить его качество и унифицировать стиль написания.

Неправильные отступы могут и не повлиять на компиляцию проекта. Это работает на читабельность и улучшение твоей производительности в качестве программиста. Но есть еще кое-что.

Например, в проекте может стоять плагин Ktlint. Это инструмент статического анализа кода (линтер) для Kotlin. Он может проверять стиль написания кода и выбрасывать ошибки, что обяжет тебя делать новый коммит и снова отправлять на ревью. Компании внедряют его, чтобы соблюдать единство стиля. И здесь мы сразу учимся писать хороший код, чтобы не переучиваться.
